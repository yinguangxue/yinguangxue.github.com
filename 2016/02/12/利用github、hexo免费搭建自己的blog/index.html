markdown语法

虽然之前我稍微看过markdown的语法，也知道只是一种更易于写blog的标记语言而已。但是要是真的让我完全开始写blog，难免有些效果不知道怎么标记。索性就通过不断写blog来强化吧。具体的教程可以参考Wow!Ubuntu

github

首先你先申请一个github账号，然后建立一个仓库，名字要和用户名一致。这个仓库就是你博客编译之后要存放的位置。假设每次编译好博客都要自己push的话，实在是太麻烦了，所以你需要先安装一个git的客户端，这个客户端能够自动的和你的github仓库建立连接。我推荐使用msysGit，Cygwin的话要安装的东西太多，在windows下还是算了。

下载并安装msysGit

配置Git

1.配置你的用户名和电子邮箱

$ git config --global user.name "yourname"
$ git config --global user.email youremail@example.com

2.配置GitHub SSH
首先需要生成SSH密钥

$ ssh-keygen -t rsa -C "youremail@example.com"

然后将.ssh文件夹里的pub文件中的内容复制到github中。
更多Git的相关学习请参考这里

hexo

安装node.js

安装Hexo

利用刚刚安装的msysgit，在目录右键，选择git bash，执行命令

$ npm install -g hexo

然后新建自己blog的文件夹，比如 D:\hexo,在这个文件夹下，打开git bash，键入

$ hexo init

之后会提醒你，don’t forget to run ·npm install· 所以下个命令

$ npm install

现在你有个空的blog，当然还有许多配置需要更改，但是现在你已经有个blog了，可以在本地先预览一下，启动命令

$ hexo server

当然，默认的端口是4000，我这个端口被占用了，所以总是不能显示，好在你可以指定端口

$ hexo server -p 2222

然后打开浏览器输入 localhost:2222就可以了。
然后怎么讲编译好的blog放到github上呢？你需要到刚刚新建的文件夹里（D:\hexo）中的_config.yml文件，找到这一行

# Deployment
## Docs: http://hexo.io/docs/deployment.html
  deploy:
  type:

并修改为：

# Deployment
## Docs: http://hexo.io/docs/deployment.html
deploy:
  type: git
  repository: git@github.com:yinguangxue/yinguangxue.github.com.git
  branch: master

然后输入

hexo deploy

过几秒钟，刷新github个人blog地址就可以看到自己的blog，当然是个崭新的待配置的，但是你已经有blog了不是吗：）
